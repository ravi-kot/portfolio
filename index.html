AI Research Engineer — Support-Aware Vision Classifiers

USC Information Sciences Institute | August 2025 – Present | Los Angeles, CA

Building reliability-aware computer vision systems that reduce false accepts, detect unsupported predictions, and abstain before unreliable decisions propagate downstream.

  • Problem: Standard image classifiers are usually forced to assign every input to one known class, even when the input is ambiguous, corrupted, or outside the training distribution. In production AI systems, this creates the visual equivalent of hallucination: a confident prediction without sufficient evidence.
  • What I built: I developed a support-aware classifier head for closed-set computer vision. Instead of trusting only a standard neural-network classifier score, the system checks whether a prediction is supported by stable evidence patterns in the image embedding. When support is weak or the prediction margin is unreliable, the model rejects the prediction rather than producing a high-confidence false accept.
  • Key result: Reduced false accepts on out-of-distribution inputs from 16.14% to 0.37% while improving traffic-sign classification accuracy from 84.99% to 85.49%.
  • Why it matters: Modern AI systems increasingly connect model predictions to automation, monitoring, and decision-support workflows. In these settings, the risk is not only a wrong prediction, but a confident wrong prediction that gets accepted and acted on. Reliability-aware classifiers expose uncertainty before errors propagate into safety-critical, medical, or enterprise systems.
Technical Details

Approach: I built the classifier head from frozen image embeddings rather than retraining a standard neural head with gradient descent. The method identifies groups of embedding dimensions that consistently support each class, stores class-specific evidence patterns, and compares each new input against those patterns before accepting a prediction.

Deployment version: The full evidence-scoring method is interpretable but more expensive than a standard classifier head. To make it deployable, I compressed the scoring behavior into a single matrix multiplication using closed-form regression. This preserves the support-aware behavior while making inference comparable to a standard neural classifier head.

Datasets: Evaluated on a 101-class object benchmark with 867 test images, a German traffic-sign benchmark with 12,630 test images and 43 classes, an aircraft-variant benchmark with 3,333 test images, and a 100-class object benchmark with 10,000 test images.

Evaluation: Measured top-1 accuracy, accepted-wrong count at 95% coverage, false positive rate on out-of-distribution inputs, risk-coverage behavior, head latency, throughput, and model-memory footprint.

Engineering: Built Python evaluation pipelines with validation-calibrated reliability thresholds. Exported C/CUDA inference runtimes for reproducible benchmarking. Created an interactive dashboard to compare standard neural classifiers against support-aware training-free classifier heads.

Limitations: The method performs best when the frozen vision backbone produces well-separated embeddings. On harder datasets with overlapping class structure, the gains were smaller. The next step is integrating reliability scores into backbone fine-tuning and agent-facing decision policies.

ML Research Engineer — Medical Imaging Pipeline

USC Big Data in Bio-transport Centre | March 2025 – Present | Los Angeles, CA

Building an AI system that analyzes retinal images for early disease detection, with results that doctors can verify.

  • Problem: Retinal images can reveal early signs of diabetes and glaucoma, but AI systems often fail when images come from different hospitals with different cameras. Doctors also need to understand why the AI flagged something.
  • What I built: A three-stage pipeline: (1) locate the optic disc landmark in the image, (2) segment its precise boundary, and (3) classify disease type with visual explanations showing which image regions influenced the decision.
  • Key result: The segmentation model maintains 94-97% accuracy when tested on hospital datasets it was never trained on — critical for real-world deployment where every hospital's images look different.
  • Why it matters: Secured $50K seed funding. A system that generalizes across hospitals and shows its reasoning is more likely to be trusted and adopted by clinicians.
Technical Details

Architecture (ViT-WaveCAM): A 3.42M-parameter segmentation model combining vision transformer attention blocks with wavelet decomposition for preserving fine boundary details. The model learns a spatial prior for where the optic disc typically appears, helping it focus on the right region.

Pipeline stages:

  • Stage 1: YOLOv11s localizer identifies optic disc bounding box (0.823 mean IoU on 1,197 test cases)
  • Stage 2: ViT-WaveCAM segments the optic disc boundary at 512x512 resolution
  • Stage 3: Disease classifier uses segmentation + vessel maps with Grad-CAM explanations
97.8% Dice (in-domain)
94.3% Dice (cross-hospital)

6 Datasets

Training data: Combined three public retinal datasets (IDRiD, DRISHTI-GS, REFUGE) with different cameras, resolutions, and annotation styles. Tested generalization on three held-out datasets (DRIONS-DB, PAPILA, MESSIDOR) without any fine-tuning.

Explainability: Beyond visual heatmaps, I built quantitative attribution metrics that measure how much the classifier relies on vessel regions vs. optic disc regions. On the FIVES dataset (4 disease classes, 200 test images), achieved 86.0% accuracy with interpretable per-class attribution patterns.

Loss function: Composite of Dice loss (region overlap), binary cross-entropy (pixel calibration), and boundary-aware loss (contour sensitivity). Boundary loss is ramped up after warmup to avoid destabilizing early training.

Limitations: Segmentation quality drops when the localizer crops poorly. The detector-to-segmenter handoff is a failure mode. Next step is joint optimization or confidence-aware cropping.

  • Problem: Standard TCP congestion control collapses under high latency and packet loss conditions common in cloud and mobile networks. Throughput drops from megabits to kilobits.
  • What I built: Custom congestion window update policy in the Linux kernel. Built a reproducible testbed using tc/netem for network emulation and iperf3 for throughput measurement.
  • Result: Improved throughput from 0.03 Mbps to 4.9 Mbps (160x) at 200ms RTT with 20% packet loss. Reduced loss-induced collapse by 42%.

High-Performance Matrix Multiplication — CUDA Optimization

Implemented optimized GEMM kernels approaching cuBLAS performance through systematic optimization.

  • Problem: Matrix multiplication is the core operation in deep learning. Understanding how to optimize it teaches GPU architecture, memory hierarchies, and performance engineering.
  • What I built: Progressive CUDA GEMM implementations: naive -> shared memory tiling -> register blocking -> vectorized loads. Benchmarked each optimization's impact.
  • Result: Achieved significant speedups through each optimization stage. Final version approaches cuBLAS performance while being fully interpretable.
Technical Details

Optimizations implemented:

  • Shared memory tiling to reduce global memory bandwidth
  • Register blocking to increase arithmetic intensity
  • Vectorized loads (float4) for memory coalescing
  • Bank conflict avoidance in shared memory

Profiling: Used Nsight Compute to analyze memory throughput, occupancy, and instruction mix at each optimization stage.

Stack: CUDA, Nsight Compute, cuBLAS (for baseline comparison).
